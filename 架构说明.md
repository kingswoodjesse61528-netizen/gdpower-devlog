# 架构说明 · 双机互为主备 / 飞书推送去重

> 适用：mini（主机/primary）+ Air（备机/backup），两机推同一个飞书群。
> 目的：**平时不重复推送 + 主机缺席时备机自动顶上**。
> 配套文档：运维细节看 `CLAUDE.md`，开发历程看 `DEVLOG-mini.md`。

---

## 1. 要解决的问题

mini 和 Air 跑同一套预测程序，都往**同一个飞书群**（同一个 webhook）推消息。两台都推 → 一部手机每天收两条重复（预测图 / 对比图 / 金山同步成功 / 同步核对，各重一遍）。

矛盾目标：① 平时只让一台推（不重复）；② 一台挂了另一台自动补（不漏）。

## 2. 核心思路：把「飞书群」当成两机的共享状态

两机**无共享盘、无共享 git**，无法直接通信。但它们都连着**同一个飞书群**——推送结果就落在群里。

> 关键：「主机今天推过了吗」这个状态，直接去群里读一眼就知道。

所以去重不靠两机互联，只靠：**备机在推之前先读群——今天有这条了就跳过，没有就补上。**

## 3. 两个飞书身份（勿混）

| 身份 | 配置键 | 能发 | 能读群 |
|---|---|---|---|
| 自定义机器人 webhook | `FEISHU_WEBHOOK` | ✅ | ❌ 永远不能 |

## 4. 决策大脑：`should_push(marker)`（在 kdocs_sync.py）

每处推送前先调 `should_push('某类')`，按本机 `NOTIFY_ROLE` 决定：

```
silent  → 永不推
primary → 直接推（主机带头，不读群）
backup  → 读群里今天的消息：
            已有这条     → 跳过（主机推过了）
            没有         → 顶上推（主机疑似缺席）
            读不到群     → 照推（fail-open，宁重勿漏）
```

**怎么认「这条」**：用**与机器标签无关的关键词子串**匹配群消息。`[Air]`/`[mini]` 发的同类消息都含同一个词，所以备机看到任意一台推过即算「已有」。

四类推送 ↔ 四个 marker：

| 推送 | 来源 | marker |
|---|---|---|
| 金山同步成功 | `kdocs_sync.notify_success` | `金山同步成功` |
| 次日预测图 | `tools/notify_prediction.py` | `次日日前电价预测` |
| 预测 vs 实际对比图 | `tools/notify_compare.py` | `预测 vs 实际` |
| 每日同步核对 | `tools/verify_sync.py` | `同步核对` |

读群实现：`_feishu_today_message_contents()` 拉今日本群所有消息的 `body.content`（原始 JSON 串，标题/正文都在里面），子串匹配即可，无需逐层解析。

## 5. 角色分工

- **mini = primary**：常年开机的台式机，主力。`should_push` 恒为「推」，不读群（稳、不受读群权限/网络影响）。
- **Air = backup**：笔记本会休眠，替补。每次推前先读群判断。
- 谁主谁备 = `.secrets.json` 里 `NOTIFY_ROLE` 一个字符串，可随时对调。

## 6. 错峰（机制成立的时间前提）

```
mini(主)  update 11:10 / verify 11:30   ← 先跑，先把消息落进群
Air (备)  update ~12:00 / verify ~12:20  ← 后跑，读群时已能看到 mini 的消息 → 跳过
```

备机必须比主机晚 ~45min。否则备机读群那一刻主机还没推，备机会误判「没人推」→ 也推 → 重复。

## 7. 日常三场景

- **A 正常日（都在线）**：11:10 mini 推 `[mini]` → 12:00 Air 读群发现都有 → 全跳过。群里每类一条。✅
- **B mini 缺席**：11:10 无人推 → 12:00 Air 读群发现空 → Air 顶上推 `[Air]`。消息没断。✅
- **C Air 缺席**：11:10 mini 照推 `[mini]`；Air 没跑无所谓。✅

## 8. 保险设计

- **fail-open**：备机读群失败时选择照推而非沉默（漏一条预测比多一条更糟）。
- **`_SESSION` 绕代理**：mini 有偶发 503 的系统代理 `127.0.0.1:1082`；kdocs_sync 所有飞书请求走 `requests.Session()`+`trust_env=False` 直连绕过，避免定时跑掉图。
- **marker 独特**：四个 marker 互不包含，也不会误匹配群里「科学上网正常」等无关消息。
- **双目录修复**：`update.sh` Step1 跑 `$GDPOWER_DIR/kdocs_sync.py` 权威副本（历史上曾误跑 WORK_DIR 旧副本）。

## 10. 一句话总结

两机不直接通信，把**飞书群当共享黑板**：主机闷头推，备机推前先看一眼「主机推了吗」——推了就让、没推就顶；再靠**错峰**保证备机看的时候主机已推完。于是平时只出一条、主机缺席自动补位。
